A) Framework - top

We have made a combination between the HTML Boilerplate, 320andup and 960 liquid grid frameworks to accomplish complete flexibility of the layout and cover all kinds of displays. You can check out the frameworks' web sites for more information. Links are bellow in Sources and Credits section.

B) Versions - Layouts, Skins - top

Layouts
The template comes in the following layout versions. You can test all available layouts and versions on the template demo.

  • Responsive version – all the way down to mobile. Located in the main folder of the download file. Remove folders fluid and fixed if you want to use just responsive.
    • Mobile – this version covers 320px to 768px width.
    • Tablet and Netbook – 768px to 1024px
    • Desktop – we have created two versions here - 1024px to 1440px, 1440px and up
  • Fixed version - 960px fixed. Located in the fixed folder of the download file.
  • Fluid version – the size of your screen. Located in the fluid folder of the download file.

Need to create your own skin, no problem. One of the fastest ways to create a new skin is:

  • Create a header and a footer image. There are .psd templates included in the download file. Unzip the PSD_files.zip and find header_footer_images.psd
  • Change the background color. Go to the your color.css file and change the background property of body
  • Change the heading color of your boxes. Go to your colors.css file and change the background property of .box
  • Change the color of header search . Go to your colors.css file and change the background property of header .search
  • Change the header color for your mobile version. Go to your colors.css file and change the background property of header.mobile


C) HTML Structure - top

Header secton is nested within a <header> tag.
Content area is nested within a div with id="main".
Footer secton is nested within a <footer> tag.
The HTML files are commented.


The template has top and side navigation included. Side navigation is a block that can be left or right side inserted.
 


D) Pages, Plugins and Options - top

We have listed the pages included in this template and we have attached a short description. We strongly recommend you to check all page examples on the template's demo. The examples are inline documented in the demo and also well documented in the code.
 
Layout
We have included some examples of column combinations.

  • 2 columns – 1:2
  • 2 columns – 2:1
  • 1 column
  • Left menu + one column
  • 1 column + left menu
  • Left menu + 3 boxes - 3 column layout turns into 4 column after screen width larger than 1440px
  • 4 boxes - 3 column layout turns into 4 column after screen width larger than 1440px
Modules
  • Gallery – three image sizes are available for a gallery page – small, medium, large. They are dynamically resized corresponding to the screen resolution.
  • Calendar – this is simple and powerful calendar using jQuery FullCalendar library (http://arshaw.com/fullcalendar/). You can drag&drop events to organize, view by month, week, day.
  • Charts – bars, interactive data points, moving, pie, dounut. We have used the plugin defaults. For more information check plugin page: http://people.iola.dk/olau/flot/examples/.
  • File manager - this is a simple and powerful file manager. It has many options – folder tree, folder view, file recognition, upload file, create file, create folder, copy, paste, remove, rename, preview, resize image, view as icons or list, refresh, back, dock/undock file manager window and more. We have used elFinder library (http://elrte.org/elfinder).
  • Contacts – iphone style contacts using SliderNav library (http://devgrow.com/slidernav/). You can use quick search by letter and view contact details.
Features
  • Tables – regular and dynamic table. We are using datatables plugin - http://datatables.net/
  • Forms - We have included a lot of form elements:
    • Input fields – regular, placeholder, password, autofocus, limited chars, tooltip, autocomplete
    • Checkboxes
    • Radio buttons
    • Text area – regular, simple, Autoresizable
    • Select fields – dropdown, multiselect
    • File uploader – regular, advanced
    • Spinners – simple, decimal, currency, UL list
    • Filtering data rows – only numbers and uppercase, only letters, only letters uppercase, custom regular expression, no space, only letters lowercase
    • WYSIWYG Editor
    • Validations – email, password, URL, min max symbols or numbers
  • Icons, Buttons, Breadcrumbs
    • 20 custom icons – 16x16 and 32x32
    • 170 jQuery UI icons – 16x16 (http://jqueryui.com/themeroller/)
    • 2400 fatcow icons - 16x16 and 32x32 (http://www.fatcow.com/free-icons)
    • Buttons – only icons, with text, colored, small, medium, large
    • Breadcrumbs – we have included a dynamic breadcrumb using this plugin here http://comparenetworks.com/developers/jqueryplugins/jbreadcrumb.html And we have styled 4 static breadcrumbs.
  • Interface elements – we have included some jQuery UI interface elements, notifications, dialogs, tabs and accordions and some non-jQuery stuff.
    • Square boxes
    • Progress bars
    • Date picker – regular, inline
    • Color picker
    • Sliders – increments, ranges
    • Ajax loaders
    • Equalizer
    • Messages, dialogs, alerts, tooltips
    • Tabs, accordions and wizard
  • Typography – headings h1 to h6, ordered unordered lists, image styles in a paragraph, link colors, grid examples
  • Error pages – we have included ready to use error pages – 403, 404, 405, 500, 503

E) CSS Files and Structure - top

CSS files are located in a folder named css. There are separate well named files for different plugins and different purposes. All css files are documented. For optimization purposes, on different pages you create we recommend loading only the files that you need.

Here are the files in the main css folder:

styles.css – this is the main CSS file.

320.css – this file is used for mobiles, supporting screen widths from 320px to 768px.

768.css – this file is used for tablets, supporting screen widths from 768px to 1024px.

1024.css – this file is used for desktops, supporting screen widths from 1024px to 1440px.

1440.css – this file is used for desktops, supporting screen widths from 1400px and up.

fixed.css – this file is used for the 960px fixed template version.

fluid.css – this file is used for the fluid template version.

grid.css – 960 grid system classes are located here.

print.css – if you need to create a print version of a page this might be of use.

reset.css – this is used to reduce browser inconsistencies in things like default line heights, margins and font sizes of headings, and so on. Read more here http://meyerweb.com/eric/tools/css/reset/


In the folder uitheme you can find the jQuery UI CSS Framework 1.8.16 css file

In the folder plugins are the css files for all the plugins. We recommend including only the files you need.

The folder named colors contains all the ready to use skins that come with this template. In the colors folder you will find a separate folder for each skin. Each skin folder for example color1 includes a css file, header and footer images. More information on how to change a skin or how to create a new one read in the Versions - Layouts, Skins section above.

F) JavaScript - top

Most of the javascripts that we have used in this template are popular jquery libraries or open source scripts. We haven’t made any changes to those libraries except the front end. Check the list Sources and Credits for more information, settings and customization. The libraries are located in the following folders /js, /blueimp, /filemanager, /uploads.
